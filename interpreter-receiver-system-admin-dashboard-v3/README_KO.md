# 행사장 통역기 대여·반납 시스템

Netlify와 Supabase를 이용한 통역기 운영 프로그램입니다.

## 포함된 기능

- 회원 사전 신청
- 개인 QR 티켓 자동 생성
- 현장에서 회원 QR 스캔
- 통역기 번호 연결 및 대여 처리
- 중복 대여 방지
- 이미 대여 중인 통역기 번호 차단
- 통역기 번호로 반납 처리
- 정상 반납·파손·분실 구분
- 언어별 신청/수령/대여 현황
- 미수령 회원 목록
- 현재 대여 중인 회원 목록
- 통역기 번호별 상태판
- 미수령자 CSV 저장
- 5초 자동 갱신 상황판
- 행사 및 통역기 번호 범위 생성

## 운영 흐름

1. 회원이 행사 전에 `/register?event=행사코드`에서 신청합니다.
2. 신청 완료 후 개인 QR 티켓이 생성됩니다.
3. 행사장에서 스태프가 `/staff`에 로그인합니다.
4. 회원 QR을 스캔하고 지급할 통역기 번호를 입력합니다.
5. 행사 종료 후 통역기 번호를 입력하여 반납 처리합니다.
6. 관리자는 실시간 상황판에서 대여 수량과 미반납자를 확인합니다.

## 1. Supabase 데이터베이스 설치

Supabase 프로젝트를 연 후:

`SQL Editor` → `New query`

아래 파일 전체를 붙여넣고 실행합니다.

`supabase/schema.sql`

이 SQL은 테이블, 중복 방지 규칙, 대여/반납 처리 함수, 보안 설정을 함께 생성합니다.

## 2. Netlify 환경변수 설정

Netlify에서:

`Project configuration` → `Environment variables`

다음 3개를 등록합니다. 범위는 **Functions**를 포함해야 합니다.

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `STAFF_PIN`

중요:

- `SUPABASE_SERVICE_ROLE_KEY`는 절대로 HTML이나 브라우저 JavaScript에 넣으면 안 됩니다.
- 이 프로젝트에서는 서비스 키가 Netlify Functions 안에서만 사용됩니다.
- 행사마다 `STAFF_PIN`을 변경하는 것이 좋습니다.

## 3. 배포

GitHub 저장소에 프로젝트를 올린 후 Netlify와 연결하는 방법을 권장합니다.

로컬 테스트:

```bash
npm install
npm run dev
```

Netlify CLI로 직접 배포:

```bash
npx netlify-cli deploy --prod
```

## 4. 첫 행사 생성

배포 후 다음 주소를 엽니다.

`https://내도메인/staff`

스태프 이름과 `STAFF_PIN`을 입력한 뒤 **행사 설정** 탭에서 입력합니다.

예시:

- 행사 코드: `SA-GERMANY-2026`
- 행사명: `Success Academy Germany`
- 행사 날짜
- 행사 장소
- 통역기 시작 번호: `1`
- 통역기 마지막 번호: `200`
- 번호 자릿수: `3`

그러면 `001`부터 `200`까지 자동 생성됩니다.

## 5. 회원에게 전달할 신청 주소

`https://내도메인/register?event=SA-GERMANY-2026`

회원은 아래 정보만 입력합니다.

- 8자리 회원번호
- 이름
- 이메일 또는 전화번호
- 통역 언어
- 반납 동의

신청 완료 후 QR 티켓이 표시됩니다.

## 주요 주소

- 회원 신청: `/register?event=행사코드`
- 회원 QR 티켓: `/ticket?t=개인토큰`
- 스태프 운영: `/staff?event=행사코드`

## 현장 권장 장비 구성

- 대여 데스크: 스마트폰 또는 태블릿 1~2대
- 반납 데스크: 스마트폰 또는 태블릿 1~2대
- 관리자 상황판: 노트북 또는 대형 모니터 1대
- 모든 통역기: `001`, `002`, `003` 형식의 번호 스티커 부착
- 가능하면 통역기에도 `RECEIVER:001` 형식의 QR 부착

## 현재 버전의 보안 구조

- 데이터베이스는 브라우저에서 직접 접근하지 않습니다.
- 모든 DB 작업은 Netlify Functions를 통합니다.
- 스태프 기능은 PIN 확인 후 사용할 수 있습니다.
- Supabase 테이블에는 Row Level Security가 활성화되어 있습니다.
- 서비스 역할 키는 서버 환경변수에만 저장됩니다.
